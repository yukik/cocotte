ワークスペースとは

ワークスペースは、ウインドウやドックが表示される領域を示します。

ここでは概要だけを説明します。より詳しい内容はWorkspace APIを確認してください。

同じユーザーが同じ所属でログインした表示端末ではすべて同じワークスペースを使用します。
表示状態は、常にサーバ側で管理されているため、同じワークスペースを使用している端末同士では操作した内容が同期されます。

デザインの変更

ワークスペースのデザインは自由に変更する事ができます。
既定のデザインでは、素早くウインドウを開く為のアイコンやメニューなどは存在していません。
利用者がよく使用するウインドウを呼び出しやすいようにデザインを変更する事で、利便性を高める事ができます。

スクリプト

ワークスペースにはクライアントサイドで使用するスクリプトを設定する事が出来ます。
設定されたスクリプトはワークスペースのテンプレートだけでなく、すべてのウインドウ・ポートレットから呼び出す事が出来ます。
cocotteでは予めよく使用するスクリプトが用意されています。

ログアウトの機能

ワークスペース自身にはログアウトの手段はもっていません。 ログアウトは/logoutへのリンクを追加する事で実装出来ます。
初期のデザインの右下に扉のアイコンにはリンクが設定されています。
