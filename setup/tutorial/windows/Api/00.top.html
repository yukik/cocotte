このウインドウは、システム開発者を対象にしています

cocotte(ココット)は、業務システム構築に特化したフレームワークです
実際に業務システムを開発する為に必要となる知識とそのAPIを説明します

コンセプト

いつでもどこでも欲しい情報をすぐに入手できる、それをすぐに作れる！

概要

会社経営には情報の取得・処理はスピードの求められる時代になりました
規模の大小に関わらず、ますます情報システムの活用を積極的に利用しなければなりません
しかしシステム開発を利用できるまで、また利用出来たとしても、なかなかシステムが経営判断に影響を与える事ができない事が多くあります。
利用者と開発者との業務知識に差が存在し、仕様の作成・開発・システムテストに時間がかかるため、どうしてもタイミングが遅れてしまいます。
本来システムは、業務に必要になった時にすぐに利用出来るようにならなければなりません。

cocotteは潜在的に感じている情報ロスのリスクを回避するために、システムの開発・提供方法を再設計しました。
多くのフレームワークや自動化ツールは開発時間を短くする事が目的になっています。
cocotteはもちろん開発工数の削減も実現できます。それ以上に理想の業務システムの提供を可能にします
ここからはいくつかの特徴にわけcocotteを説明していきます。

node.js+Express+cocotte

cocotteはnode.jsをサーバサイドのプログラム言語に採用しています。
node.jsはクライアントサイドの事実上標準であるjavascriptをサーバサイドで記述する事が出来ます。
javascriptを学習するだけでバックエンドもUIも開発出来る利点があり、開発者の負担を減らします。
また、定番のフレームワークであるExpressをベースに業務システムに必要な機能をcocotteが補っています。
expressは非常に優れたフレームワークではありますが、いざ業務システムを作成するとなると自由度があるため、 保守が困難になる可能性があります。
cocotteは単なるjavascriptのライブラリ群ではなく、業務システムを作成する上での規約を明確にする事で システムリリースのその後の保守・カスタマイズコストを削減します

進化型プロトタイピング開発手法

通常、システム開発のワークフローは要件定義・外部設計・内部設計・開発・テストと段階的に行っていきます。 ウォータフォールでは発注者は開発が開始されるまで具体的に動作する画面を確認する事が出来ません。 そのため、開発者は外部設計の段階でモックアップなどを提供し、発注者との共通認識を持つ事になります。
アジャイルも基本はウォータフォールと同じですが、動作するソフトウェアを優先するため、モックアップの作成は省略される場合があります。

cocotteもアジャイル開発に似ていますが、さらに開発期間を短くし利用者との１つのミーティングのあとすぐに開発を行います。
短い打ち合わせの度に実際に動作する画面を少しずつ提出する開発手法となります。
部分的に進められる開発は優先して進める事で、発注者との認識の相違が極力抑えられる利点があります。
（たとえ不完全でも）動作する画面を提供するため、cocotteでは「進化型プロトタイピング開発手法」となります。
アジャイル開発は小さなウォーターフォールを短期間（２週間程）で繰り返していきますが、 「進化型プロトタイピング開発手法」はそれよりも短い期間（最長でも３日）で発注者に画面を提供する事を実現させます。
もちろん、複雑な機能を持つ画面をすぐに作成して提供する事はできません。
しかしHTMLを記述することができる担当者がいれば、まだ動作しない画面も部分的に動作しないが画面を確認する事ができるようにします。
これによりExcelモックアップなどのドキュメントを作成しません。

マルチウインドウ

マルチウインドウはWindows/Mac/Linuxでも採用されている標準的なGUIです。
cocotteは多くの方が慣れ親しんだGUIをブラウザ上に実現した完全なSingle Page Application(SPA)です。
SPAの特徴は、ユーザーが情報の優先順位をつけて自由に画面上に配置する事が出来る事です。
業務システムは多機能になるにつれ、すべての利用者が納得する画面を作成する事が難しくなります
私たちは、マルチウインドウにすることで多くの画面の問題は解決すると考えています。

ポートレットによるモジュール化

業務システムを作成する上で要件が、明らかに案件に左右されない共通の機能を利用する事で 実現出来る事があります。cocotteでは共通の機能を簡単に呼び出す事ができる方法を提供しています。
ポートレットと呼ばれる共通の機能は、利用するだけではなく開発者自身が自由に追加・改変する事が出来ます。

リアルタイムデータ処理

長らくWebアプリケーションの共通の問題として、「表示されたデータはそのときから過去の情報になる」事が挙げられます。 cocotteでは、値の更新をリアルタイムで他のシステム利用者も確認出来るように設計されています。
リアルタイムデータ処理には多大なコーディングが必要になっていましたが、cocotteでは従来のステートレスの開発手法と 変わらないコーディングでリアルタイムデータ処理を実現する事が出来ます。

状態保存

cocotteでは、ログイン・ログアウトによって画面の状態がリセットされる事はありません。
些細な事と思われるかもしれませんが、常に最適に配置されたウインドウの状態であり続ける事は、 毎日使用する利用者の時間短縮に貢献します。

開発を容易にするリレーション機能

データ同士の連携は、多くなる毎に開発・テストの工数が大きくなる要因です。
cocotteでは、異なるウインドウ同士でも簡単に連携する事が出来るダイアグラムと呼ばれる機能を提供しています。
ダイアグラムを使用する事で大きく２つの利点があります。
（１）一つのウインドウの操作だけをコーディングするだけで、矛盾なく他のウインドウの情報を”自動で”更新します。
（２）テーブル間の整合性を保つ事ができます。
