ファイル型

ファイル情報はファイル実体の更新と追記情報の更新と２回のタイミングで実行され
ここでのvalidは追記情報の更新

ファイル情報を保存・取得出来るデータ型
複数のファイルを格納する事が出来る
ファイル情報
	name       : ファイル名（アップロード時のファイル名）
	description: ファイル内容の簡易な説明
	uri        : ファイルの実体にアクセスする為のURI
	thumbnail  : 画像ファイル、映像ファイルのサムネイルが存在するか。trueの場合は同じフォルダにサムネイルファイルが作成される
	type       : ファイルのタイプ
	size       : ファイルサイズ
	upload     : アップロード日時
	user       : アップロード作業者
	history    : 過去ファイルの情報 [{}]

schema
	limit    : ファイル数の上限。既定値:undefined (無制限)
	type     : ファイルタイプの制限。既定値:undefined　(制限なし)
	size     : ファイルサイズの上限(MB)。既定値:100
	history  : 保管する世代ファイルの数。既定値:3

schema (binding時)
	filtering : 絞り込み予定値
	filtered  : 現在の絞り込み
