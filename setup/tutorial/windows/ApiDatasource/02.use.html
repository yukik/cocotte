データソースを作成して使用するにはいくつかの方法があります。

コンストラクタ(schema)
コンストラクタを使用した場合のDatasourceの例は以下の通りです
var Datasource = require('cocotte/datasource');
var schema = {
    name  : 'student'
  , fields: {
                name : {type: 'text'  , caption: '氏名'}
              , score: {type: 'number', caption: '点数'}
            }
};
var ds = new Datasource(schema);
コンストラクタ(model)
コンストラクタにmodelオブジェクトを引数として設定した場合は、フィールド情報などが 自動設定され、永続化の機能をもったデータソースを簡単に作成する事が出来ます
var Model = require('cocotte/model')
  , Datasource = require('cocotte/datasource');
var schema = {
    name  : 'student'
  , fields: {
                name : {type: 'text'  , caption: '氏名'}
              , score: {type: 'number', caption: '点数'}
            }
};
var md = new Model(schema);
var ds = new Datasource(md);
モデルのスキーマとデータソースのスキーマは似ていますが、異なる部分があるので 詳しくはそれぞれのAPIを参照してください。
Datasource.create(schema)
var Datasource = require('cocotte/datasource');
var schema = {
    name  : 'student'
  , fields: {
                name : {type: 'text'  , caption: '氏名'}
              , score: {type: 'number', caption: '点数'}
            }
};
var ds = Datasource.create(schema);
先ほど異なるのはnew DatasourceのかわりにDatasource.createを使用しただけですが、 変数dsのスコープが破棄されても、作成されたデータソースは破棄されません。
次の方法で、いつでも呼び出す事が出来ます 
var dsId = ds.id;
var ds2 = Datasource.get(dsId);
dsとds2は同じデータソースを示します。
データソースを破棄するには、次のように行います 
Datasource.remove(ds);
Datasource.create(model)
new Datasource(model)と同様にDatasource.create(model)も動作します。
Datasource.create(builtInName)
予め定義されていたスキーマ（ビルトイン）を使用すると簡単にデータソースを作成することが出来ます。
