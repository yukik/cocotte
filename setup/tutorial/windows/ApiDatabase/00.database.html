データベースへの永続化の処理を担当します
設定ファイルは接続情報を保有します。
作成するにはdatabasesフォルダに接続名と同名のファイルを作成してください。
default.jsのみ必ず作成する必要があります。
default.jsに設定された接続情報は、アプリケーションの構築情報やセッション情報の格納、データベースの指定のないモデルの接続先に設定されます。

作成方法

初期設定で唯一作成されているdefault.jsを以下に記述します 

var db = {
    driver  : 'mongodb'
  , host    : '127.0.0.1'
  , port    : 27017
  , db      : 'cocotte'
  , readonly: false
  , file: {
        type  : 'file'
      , path  : 'files'
      , subdir: 'daily'
    }
};
module.exports = exports = db;

プロパティ

上記のファイルに記載されている設定を解説します

driver
ドライバ。現在対応しているデータベースはMongoDBのみのため、mongodb固定です
host
接続先
port
ポート番号
db
データベース
readonly
データの取得のみ許可されている場合はtrueにします。 default.jsでは必ずfalseに設定してください。 書込みの出来ないデータベースはdefault.jsに設定出来ません。
file.type
ファイルを保存する際にどのように保存するかを設定します。 現在は物理保存のみに対応していますので、file固定です。
file.path
物理保存されるファイルをどのディレクトリに設定するか。 アプリケーションルートのフォルダからの相対パスです。
file.subdir
サブディレクトリを作成する場合の設定です。 項目なしもしくはnullを設定するとサブディレクトリは作成されません
現在はdailyのみに対応しています。 dailyは日付毎にサブディレクトリが作成されます
