ウインドウの開き方

ウインドウを開く為には、エレメントにイベントを追加する方法と、スクリプトでnew cocotte.Win(options)を記述する方法と２つの方法から行う事が出来ます。
前者をイベント方式、後者をスクリプト方式とします。 イベント方式はビルトイン（予め定義されたウインドウ設定）から名称を選択してウインドウを開きます。 スクリプト方式はビルトインに限らずウインドウを作成して開く事が出来ます。
それぞれの設定方法は次の通りです

イベント方式

ウインドウ内のエレメントにイベントを追加するには、data-winname='ウインドウ名'の属性を追加してください
ビルトイン名が一致するウインドウを開きます。 指定したウインドウがビルトインに登録されていない場合は、ウインドウを開きません。
ウインドウを開く際にさらに様々オプションを設定する事が出来ます。オプションは同様にdata-winを接頭語にもつ属性をdata-winnameと同様に記述してください。
ただし、イベント方式では指定出来ないオプションが存在します。説明にて確認してください。 オプションはは後述します。
以下に、名称'tutrialMap'という名称のウインドウを最大化状態で表示する例を示します 

<a data-winname='tutorialMap' data-winmode='maximize'>地図を開く</a>
実行：地図を開く
イベント方式で開かれたウインドウは、他の端末でも同期されます。
また、ウインドウを開くイベントを発行したエレメントの情報から出来るだけ関連した情報を表示するように自動的に設定されます。

スクリプト方式

スクリプト方式で開いたウインドウは、他の端末に同期されません。
そのため、一時的なウインドウの表示に使用します。
クライアントサイドのいずれかの処理で次の様に記述してしてください。

var title    = 'タイトル'
  , contents = '内容'
  , options  = {};
new cocotte.Win(title, contents, options);
title, optionsは省略する事が出来ます。
contentsのみ必須です。 引数が2つの場合に第２引数がObject型の場合にtitleをcontentsに、contentsをoptionsに設定します。

引数およびオプション

title
イベント方式では指定出来ません ウインドウのタイトルバーのテキスト
contents
イベント方式では指定出来ません コンテンツを指定します。 文字列の場合はHTML。その他の指定はtextとiframeのみで、portletをオプションで指定する事は出来ません。 ポートレットはビルトイン名を指定したときのみ利用可能です
name
イベント方式では必須項目です。
ビルトイン名を指定します。
ウインドウを検索する場合に使用します。
mode
ウインドウを表示させる際の状態を指定します。
normal
ウインドウを通常の状態で表示します。既定値
maximize
ウインドウを最大化の状態で表示します
minimize
ウインドウを最小化の状態で表示します
dialog
ダイアログウインドウとして表示します。ダイアログウインドウを開いている間は、他のウインドウを操作する事は出来ません。
最大化、最小化を行う事は出来ません。
info
情報ボックスとして表示します。情報ボックスはウインドウの外側をクリックした場合に閉じられます 最大化、最小化を行う事は出来ません。
template
テンプレート名。
ウインドウの外枠のデザインを選択します。
予め/public/javascript/template.jsにテンプレート名を登録する必要があります。
指定した名称のテンプレート名が存在しない場合は、modeにより次のように設定されます。
  • normal,maximize,minimize --> defaultTo
  • dialog --> dialog
  • info --> info
size
表示サイズを設定します。
配列を渡してください。[width, height]
イベント方式では、'width,height'の文字列です
position
表示位置を設定します
配列を渡してください。[top, left]
イベント方式では、'top,left'の文字列です
closable
ウインドウをイベントから閉じる事ができるかどうか？ Boolean。既定値 true。イベント形式では'false'を設定すると閉じない設定にします
find
ウインドウの検索し、存在した際にどのような処理を行うか？
文字列。既定値 auto
auto
infoのプロパティが設定されている場合は、infoの照合も行います
照合で同じとみなされるウインドウが存在しない場合は新しく開きます。
存在する場合に、さらにinfoのプロパティが完全一致かどうかを確認します
完全に一致している場合はputInを行います。
一部でも異なっている場合は次の動作を行います。
  1. (onRedrawが設定されている場合)ウインドウのonRedrawを呼び出します
    (onRedrawが設定されておらず、onDrawが設定されている場合に)ウインドウのonDrawを呼び出します
  3. コンテンツのうちポートレットの再読み込みを行う
skip
常に新しいウインドウを開きます
open
optionsの照合を行い、同じウインドウが存在しない場合は新しく開きます
同じ場合はputInの動作を行います
異なる場合は次の２つを順に行います
  1. (onRedrawが設定されている場合)ウインドウのonRedrawを呼び出します
    (onRedrawが設定されておらず、onDrawが設定されている場合に)ウインドウのonDrawを呼び出します
  3. コンテンツのうちポートレットの再読み込みを行います
optionsの照合には次のルールが設定されます
  • (optionsのプロパティにdatasourceとrownoの両方が設定されている場合)
    両方とも同じであるウインドウを照合が成功したとする
  • (optionsのプロパティにdatasourceとrownoのいずれかが設定されていない場合)
    すべてのオプションが一致するウインドウを照合が成功したとする
options
optionsはウインドウを開く際に渡される値を設定する事が出来ます。
オブジェクトを設定してください。
{
    address: '宮崎県都城市'
}
追加情報を特定のポートレットに向けて設定する場合はprefixを追加します
ポートレットの特定にはコンテンツを イベント方式では文字列またはスクリプト名を設定します
形式1 'portelName{key:fieldName,key:fieldName}'
例えば、'tutrialMap{address:jitaku_jusho}'を設定してtutrialMapを開いた場合は、{address: '宮崎県都城市'}などの値が渡される事が期待されます。
形式2 'script.scriptName'
