ウインドウに多くのHTMLコンテンツやテキストコンテンツを追加する場合は、index.jsに記述する際に煩雑になります。
別ファイルにする事で、余分なエスケープ等必要なく管理が楽になります。
ここでの解説では、別ファイルで管理する為のヘルパーを使用します

cocotte/window/helperモジュール

html
html (dir, subdir, file)
@param {String} dir 検索対象のディレクトリ
@param {String} subdir 検索対象のサブディレクトリ。省略可能
@param {String} file HTMLファイル名。拡張子省略

対象ディレクトリにあるHTMLファイルを検索します。
{html: ファイルの内容}を返します
text
text (dir, subdir, file)
@param {String} dir 検索対象のディレクトリ
@param {String} subdir 検索対象のサブディレクトリ。省略可能
@param {String} file テキストファイル名。拡張子省略

対象ディレクトリにあるテキストファイルを検索します。
{text: ファイルの内容}を返します
container
container (dir, subdir) @param {String} dir 検索対象のディレクトリ
@param {String} subdir 検索対象のサブディレクトリ。省略可能

対象ディレクトリにあるcontainer.htmlを検索します。
ファイルの内容をテキストとして返します

index.js

var helper = require('cocotte/window/helper');

var window = {
    title   : 'Window API'
  , contents: [
            helper.html(__dirname, '00.first')
          , helper.html(__dirname, '01.second')
          , helper.text(__dirname, '02.third')
        ]
  , container: helper.container(__dirname)
};
module.exports = exports = window;

container.html

<h2>コンテナ</h2>
<ul>
	<li data-script='tab' class='tabActive'>first</li>
	<li data-script='tab' class='tabInactive'>second</li>
	<li data-script='tab' class='tabInactive'>third</li>
</ul>
<div class='tab'>
	<class='tabActive'>{0}</div>
	<class='tabInactive>{1}</div>
	<class='tabInactive'>{2}</div>
</div>

index.jsがすっきりして全体が見やすくなりました。
さらにHTMLとテキストの混在でもそれぞれを別々に編集する事が可能です。 コンテナは今回タグ機能を追加したものを例にしましたが、 工夫次第で可変グリッドやアコーディンパネルなど実装する事が出来ます
helperの__dirnameはindex.jsと同じディレクトリを示します。
サブディレクトリを作成し管理する場合はhelper.html(__dirname, 'sub1', '00.first')とする事が出来ます
